# gstack 미설치 스킬 참고 목록

gstack(https://github.com/garrytan/gstack)에서 분석했지만 아직 설치하지 않은 스킬 목록입니다.
필요할 때 SKILL.md로 직접 작성하거나 gstack에서 가져와서 사용할 수 있습니다.

**설치된 스킬**: office-hours, investigate, retro, ship, cso, benchmark

---

## 전략·기획

| 스킬 | 역할 | 핵심 동작 | 언제 유용한가 |
|------|------|---------|------------|
| `/plan-ceo-review` | CEO/창업자 | "10점짜리 버전은 뭔가?"를 묻고 스코프를 확장/축소/유지로 판단 | 기능 우선순위 결정 전 |
| `/plan-eng-review` | 엔지니어링 매니저 | 아키텍처, 데이터 흐름, 상태 전이, 장애 모드, 테스트 계획 수립 | 대규모 구조 변경 전 |
| `/plan-design-review` | 시니어 디자이너 | 정보 계층, 상호작용 상태, 로딩/에러 8개 항목 설계 평가 | 디자인 검토 시 |
| `/autoplan` | 자동 파이프라인 | CEO → 디자인 → 엔지니어링을 자동 순차 검토 | 전체 플로우 한 번에 돌릴 때 |

## 설치 방법

필요한 스킬이 생기면 `.claude/skills/{이름}/SKILL.md`를 직접 작성하거나,
gstack 원본을 참고해 DS_2 맥락에 맞게 재해석합니다.

```bash
# gstack 원본 스킬 확인 (설치 없이 코드만 확인)
# https://github.com/garrytan/gstack/tree/main/skills
```
